Κλάσεις 

Η υλοποίηση βασίζεται στην αρχιτεκτονική Sockets (TCP) χρησιμοποιώντας Threads για την ταυτόχρονη εξυπηρέτηση πελατών.

Server: Η κύρια κλάση του εξυπηρετητή. Ακούει σε μια καθορισμένη θύρα (Port) και για κάθε νέα σύνδεση δημιουργεί ένα αυτόνομο νήμα (ClientHandler).
Client: Η διεπαφή του χρήστη. Δέχεται ορίσματα από τη γραμμή εντολών, συνδέεται στον εξυπηρετητή, αποστέλλει ένα Request και αναμένει ένα Response.
Account: Κλάση που αποθηκεύει τα δεδομένα του χρήστη: username, authToken και τη λίστα μηνυμάτων messageList.
Message: Κλάση που αναπαριστά το μήνυμα με τα πεδία sender, receiver, mes (body) και την κατάσταση isRead.
Request & Response: Βοηθητικές κλάσεις (Serializable) για την ομοιόμορφη ανταλλαγή δεδομένων μεταξύ Client και Server. 


Παραδοχές & Λεπτομέρειες Υλοποίησης (Assumptions) 

Μη μόνιμη αποθήκευση .
Το σύστημα δεν χρησιμοποιεί βάση δεδομένων ή αρχεία για την αποθήκευση των δεδομένων. Σε περίπτωση επανεκκίνησης ή τερματισμού του Server, όλα τα δεδομένα χάνονται !

Μοναδικότητα των Message IDs.
Χρησιμοποιείται κεντρικός μετρητής για τα IDs των μηνυμάτων. Το ID ενός μηνύματος είναι μοναδικό σε ολόκληρο το σύστημα. 

Διαχείρηση AuthToken.
Τα tokens παράγονται τυχαία στο εύρος 1000-9999. Ο μηχανισμός ελέγχει για συγκρούσεις πριν αναθέσει ένα token.
